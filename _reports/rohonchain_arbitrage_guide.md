# 📚 【推文分析：Polymarket 量化套利完整指南】

**作者**: @RohOnChain (后端开发者，专注 HFT 和量化交易系统)

## 📌 核心内容摘要
这篇文章系统性地拆解了在 Polymarket 上进行**无风险套利**的数学框架和工程实现。基于一篇真实的研究论文，揭示了专业交易者在一年内从 Polymarket 提取了 **$4000 万**的保底套利利润。

---

## 🔢 五大技术模块

| 模块 | 核心概念 | 作用 |
| :--- | :--- | :--- |
| 1. 边际多面体问题 | 整数规划 (IP) | 解决 2^63 种结果组合的暴力搜索 |
| 2. Bregman 投影 | KL 散度 | 计算最优套利头寸 |
| 3. Frank-Wolfe 算法 | 迭代线性规划 | 使投影计算可行 |
| 4. 非原子执行 | VWAP + 滑点估算 | 解决订单簿执行风险 |
| 5. 完整系统架构 | WebSocket + Gurobi | 实盘部署 |

---

## 💰 关键数据
- 顶级套利者一年赚取 **$2,009,631**（4,049 笔交易，平均每笔 $496）
- 41% 的单条件市场存在套利机会
- 中位数定价偏差：**40%**（$0.60 vs 应有的 $1.00）
- 延迟差距：专业系统 ~2s vs 散户 ~30s

## 🔥 可借鉴的思路

1.  **依赖性检测**: 用 LLM（如 DeepSeek）自动识别市场间的逻辑依赖关系。
2.  **VWAP 估算**: 不要假设能以报价成交，按订单簿深度计算实际执行价。
3.  **最低利润阈值**: 文中用 $0.05 作为执行门槛（覆盖 gas + 滑点）。
4.  **执行窗口**: Polygon 2 秒出块，所有腿必须在同一区块确认。

## 📊 与 HLP 监控对比

- 门槛更低，但利润空间也更小
